## --uitkLink CSS Custom Property API

The default foreground and background colors of the Link are provided by the theme variables. See Theme documentation for detailed usage guidance.

The CSS custom properties below are consumed by Link, but not defined by Link. They can be defined via a container or
a custom class name to override avatar styling. They will always take precedence over default styles, whether from theme variable
or declared locally. Again, see Theme documentation for guidance on when to use variables from the Custom Property API.

The CSS is as simple as below:

```css
/* from the file link.stories.newapp-link.css */

.uitk-light.uitk-newapp .uitkLink {
  --uitkLink-color: var(--uitk-color-green-200);
  --uitkLink-color-hover: var(--uitk-color-purple-200);
  --uitkLink-color-active: var(--uitk-color-green-200);
  --uitkLink-color-focus: var(--uitk-color-red-200);
  --uitkLink-color-disabled: var(--uitk-color-grey-100);
}

.uitk-dark.uitk-newapp .uitkLink {
  --uitkLink-color: var(--uitk-color-red-200);
  --uitkLink-color-hover: var(--uitk-color-green-200);
  --uitkLink-color-active: var(--uitk-color-red-200);
  --uitkLink-color-focus: var(--uitk-color-green-200);
  --uitkLink-color-disabled: var(--uitk-color-grey-50);
}
```
